Type of data collected

ScoutLog collects 3 types of data:

  • The data created directly by the user ("places data") through normal usage of the app, which includes personal locations, tags, notes, and media files.
  • The data that represents app usage ("app usage data") which is automatically generated through normal usage of the app.
  • The data that is automatically gathered when the app encounters an error ("error data"). This will not include any personal information, although for some kinds of errors, media file paths may be gathered.

Why the data is collected

Places data is collected for the normal functioning of the app. It's stored locally on the user's device and on a cloud storage platform such as Google Drive, if the user sets up data syncing. The developer never has access to this data.

App usage data is collected to help determine the most and least popular features.

Error data is collected to help debug any errors encountered by users.

Data sharing with third parties

Places data is never shared with the developer, but it will be shared with a cloud storage platform like Google Drive if the user sets up data syncing.

App usage data is collected by Google Analytics.

Error data is collected by crash reporting software called Crashlytics.

Users' right to withdraw consent

Places data is obtained through normal usage of the app, so users may withdraw consent to data collection by uninstalling the app. Data stored on the device is deleted by uninstalling, but any data stored on a cloud storage platform such as Google Drive must be explicitly deleted.

App usage data cannot be deleted, but users can withdraw consent through a setting in the app.

Error data cannot be deleted, and users may withdraw consent to data collection only by uninstalling the app.
